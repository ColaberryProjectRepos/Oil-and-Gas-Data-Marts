Problems that Occurred During the Project
  1. There were many values in the staging table that had to be replaced. Derived Column transformation was used but there was a limitation on the output column length so all values could not be processed. In order to fix this, the Case When Statement was used in SSMS to replace values.
  2. When the fact table was loaded using the Lookup transformation, and the package executed for the second time, there were duplicate records on the tables. In order to fix this, an additional Lookup transformation was used to match the surrogate keys, and the OLE DB Command transformation was used to update records in case updates were made on the source table.
